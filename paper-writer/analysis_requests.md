### Naive hard-classification baseline

- **Requested by**: paper-writer
- **What's needed**: Add a comparison estimator that treats the modal classifier prediction as the true regime and estimates regime-specific regressions without confusion-matrix correction. Report bias, RMSE, and coverage against the corrected MLE across the same Monte Carlo grid, or across a representative subset if runtime is prohibitive.
- **Why**: The first draft can validate corrected-MLE finite-sample behavior, but a naive baseline would make the correction payoff more directly visible to readers.
- **Status**: ✅ Complete (2026-04-28)
- **Output**:
  - Per-replication parquet results: `mc_naive_results/cell_*.parquet` (14,400 cells × 200 replications, full factorial grid matching `mc_coverage_results/`)
  - New figure: `paper-writer/results/figures/figure_5_naive_vs_corrected.{pdf,png}` (bias and coverage comparison)
  - Updated tables: `paper-writer/results/tables/table_A2_coverage_rho_0_6.tex` now includes a `naive_coverage` column; `paper-writer/results/tables/table_A2_coverage_all_cells.csv` has full naive coverage and SE columns.
  - Machine summary: `paper-writer/results/tables/simulation_evidence_summary.md` now includes naive headline metrics and side-by-side comparison tables.
- **Generated by**:
  - `paper-writer/scripts/run_naive_baseline.py` (new) — regenerates each MC replication with the same seed, computes hard-classification OLS with cluster-robust SE clustered by driver, saves per-cell parquet.
  - `paper-writer/scripts/generate_simulation_evidence.py` (updated) — loads naive results alongside corrected MLE/IRLS and produces the comparison artifacts.
- **Specification**: Hard classification by `argmax(misclass_regime_*)`. Each predicted regime is fit with OLS using its own drought covariate. Standard errors are Liang-Zeger cluster-robust at the driver level with the small-sample correction `G/(G-1) * (n-1)/(n-k)`. 95% CIs are `est ± 1.96 * SE`. When fewer than `p+1` observations land in a predicted regime, that regime-replication is treated as unidentified and excluded from coverage.
- **Sample**: Same Monte Carlo grid as the corrected MLE: `R in {2, 4, 10}`, `weight in {0.0, 0.1, 0.3, 0.5, 0.7, 0.9}`, `rho in {0.0, 0.3, 0.6, 0.9}`, 200 replications per cell, 200 drivers × 15 periods per replication.
- **Notes**: Headline finding — at low classification error the naive baseline matches MLE, but coverage and RMSE diverge sharply at high misclassification in multi-regime designs. For `R=4, w=0.9`, naive 95% coverage is 0.005 vs corrected MLE bootstrap coverage 0.471. For `R=10, w=0.9`, naive coverage is 0.145 vs MLE bootstrap 0.483. The comparison cleanly motivates the confusion-matrix correction.
